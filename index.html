AI-Driven Real-Time Attendance Monitoring System

By:Prabhala Sai Sirisha Devi

Department/College: CSE - Honors, KL Deemed to be University

Abstract

  • The project aims to develop a smart attendance system using AI.
  • Combines face recognition (Multi-frames), motion analysis (walking pattern).
  • Data is stored securely with offline support using cache-like memory.
  • Designed to prevent misuse and increase reliability over existing systems.

Objectives

  • Use ESP32-CAM for face data capture and recognition.
  • Sync attendance with Firebase in real time.
  • Fetch data into Django-based web interface for data access & management.
  • Enhance the system by recognizing unique human motion (walking styles).
  • Implement offline attendance support using cache-style memory instead of only cloud or physical memory.
  • Prevent misuse through multi-factor validation (gait-based + face + on-premise device).

Problems in Current Systems (Govt of AP & External Devices)

  • Smartphone misuse: Time/date manipulation, tech illiteracy.
  • Biometric spoofing: Latex fingers, proxy attendance.
  • External devices: Expensive, fixed, not dynamic.
  • No true offline support: Delayed syncing, not real-time.
  • Static face data: Cannot dynamically update face DB remotely.
  • No liveness check or motion validation in most systems.

Current Progress

  • ESP32-CAM captures and recognizes faces.
  • Attendance marked with name + time in Firebase.
  • Web app shows attendance data from Firebase.
  • Face data is static – stored locally, not in Firebase.
  • Offline face recognition not possible due to ESP32 dependency on Wi-Fi.

Planned Solutions

  • Gait – based recognition (walking patterns) & Multi-frame Face recognition to enhance uniqueness & accuracy.
  • Offline cache memory mechanism to store attendance temporarily until sync.
  • Admins can upload facial data dynamically to Firebase for comparison.

Conclusion

  • Project aims to build a more secure, smart, accurate and misuse-proof attendance system.
  • Combines face + motion + offline support to overcome current limitations.
  • Early progress is complete; next stage will involve gesture recognition model and offline data handling.
